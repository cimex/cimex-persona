So what am I?


I am:

a tool to generate random placeholder user data. Whilst “Lorem Ipsom” can deal with your empty static content area, user driven websites needs higher standard of user data to simulate. Having data in early in a new website's life cycle will increase development productivity by a long way.

Also becuase:

Showing your client development progress and convincing them that their investment is well spent is not an easy task. Nor does it bode well when your awesome user directory and social interaction demo is full of non-contextual users called 'test1, test2' etc..

Finally its for your own good because:

Spot bugs early! It's one thing outputting a 5 character name ('test 1 for example'), but how do you know what will happen to the UI component if you output a long name? How about a name with special characters? Having meaningful data means you can flag issues early and not leave it till it's too late.

Use Me


To use me in your Java application, download the core here. Remember to include the dependencies in your class path, below is a quick start: 

            for (PersonaModel model: generatePersona(100)){
                System.out.println(model);
                System.out.println(model.getFirstName());
            }

Integrate me with XML


Call http://persona.timothyyip.org/persona/xml/1 for 1 persona. Replace the 1 to get...... make a guess. There's a limit of 500 max {sorry to disappoint!}. 

        
<list>
	<org.timothyyip.randompersona.PersonaModel>
		<title>Ms</title>
		<firstName>Alicia</firstName>
		<lastName>Kinney</lastName>
		<fullName>Alicia Kinney</fullName>
		<companyName>Teradisco</companyName>
		<email>alicia.kinney@teradisco.co.uk</email>

		<male>false</male>
	</org.timothyyip.randompersona.PersonaModel>
</list>
